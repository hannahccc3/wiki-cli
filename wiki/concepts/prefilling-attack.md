# Επίθεση Προσυμπλήρωσης (Prefilling Attack)

## Επισκόπηση

Η Επίθεση Προσυμπλήρωσης είναι μια επίθεση που εκμεταλλεύεται τη δυνατότητα προσυμπλήρωσης της απόκρισης του μοντέλου με μια επιθυμητή αρχική συμβολοσειρά. Αυτή η τεχνική αξιοποιεί μοναδικές λειτουργίες API που δεν είναι διαθέσιμες σε όλους τους παρόχους LLM.

## Περιγραφή της Επίθεσης

### Βασική Ιδέα

Αντί να προσπαθήσουμε να βελτιστοποιήσουμε ένα επίθεμα για να αυξήσουμε την πιθανότητα μιας επιθυμητής απόκρισης, η επίθεση prefilling:
1. Παρέχει απευθείας την επιθυμητή αρχική απόκριση
2. Αφήνει το μοντέλο να συνεχίσει την απόκριση
3. Εκμεταλλεύεται τη φυσική συνέχεια της γλώσσας

### Παράδειγμα

```
Σύστημα: [Προτροπή ασφαλείας του Claude]
Χρήστης: [Επιβλαβές αίτημα]
Claude: Sure, here's how to [συνέχεια του μοντέλου]
```

## Εφαρμογή στα Μοντέλα Claude

### Γιατί Λειτουργεί

Τα μοντέλα Claude της Anthropic:
- **Μην εκθέτουν logprobs**: Αποτρέποντας την άμεση επαναληπτική βελτιστοποίηση
- **Υποστηρίζουν prefilling**: Επιτρέποντας τον έλεγχο της αρχικής απόκρισης

### Ποσοστά Επιτυχίας

| Μοντέλο | Με Prefilling |
|---------|---------------|
| Claude 2.0 | 100% |
| Claude 2.1 | 100%* |
| Claude 3 Haiku | 100% |
| Claude 3 Sonnet | 100% |
| Claude 3 Opus | 100% |
| Claude 3.5 Sonnet | 100% |

*Σημείωση: Το Claude 2.1 απαιτεί έως και 100 επανεκκινήσεις

## Σύγκριση με Άλλες Μεθόδους

| Μέθοδος | Claude 3 Opus |
|---------|---------------|
| Transfer Attack | 0% |
| **Prefilling Attack** | **100%** |

Η επίθεση prefilling αποδείχθηκε σημαντικά πιο αποτελεσματική από την transfer attack για ορισμένα μοντέλα Claude.

## Σημασία

Αυτή η έρευνα αναδεικνύει ότι:
- Διαφορετικά API έχουν μοναδικές ευπάθειες
- Οι προσαρμοστικές επιθέσεις πρέπει να λαμβάνουν υπόψη τις ιδιαιτερότητες κάθε πλατφόρμας
- Η απόκρυψη logprobs δεν είναι αρκετή άμυνα
