# Αντίπαλοι Ασφαλείας (Security Adversaries)

## Επισκόπηση

Οι **Αντίπαλοι Ασφαλείας** είναι αντίπαλοι που επιδιώκουν να αναγκάσουν το θύμα LLM να παράγει συγκεκριμένες απαγορευμένες εξόδους. Η επιτυχία τους μετράται **διχοτομικά**: είτε το θύμα παρήγαγε τη συγκεκριμένη έξοδο είτε όχι.

## Παραδείγματα

### Jailbreaks

Επιθέσεις που παρακάμπτουν τα φίλτρα περιεχομένου για να αναγκάσουν το LLM να δημιουργήσει επιβλαβές περιεχόμενο. Η επιτυχία αξιολογείται ελέγχοντας αν η έξοδος αρχίζει με συγκεκριμένη συμβολοσειρά (π.χ., "Sure, here's how to...").

### Prompt Injection

Εισαγωγή κακόβουλων οδηγιών στην είσοδο για τον χειρισμό του μοντέλου να εκτελέσει συγκεκριμένη εργασία.

## Μαθηματικός Ορισμός

Ο στόχος του αντιπάλου ασφαλείας χαρακτηρίζεται από μια **συνάρτηση-στόχο** t: X → P(Y) που αντιστοιχίζει εισόδους σε σύνολα απαγορευμένων εξόδων.

Η συνάρτηση βαθμολόγησης είναι:
```
s(q, a) = 1_{a ∈ T_q}
```

όπου T_q = t(q) είναι το σύνολο των απαγορευμένων εξόδων.

## Περιορισμοί

Οι αντίπαλοι ασφαλείας **δεν συλλαμβάνουν πλήρως** πολλές απειλές, όπως:

- Δημιουργία deepfakes
- Πληροφοριακές οδηγίες για παράνομες δραστηριότητες

Αυτό συμβαίνει διότι οι αντίπαλοι μπορούν να συνθέσουν μεμονωμένες αποδεκτές εξόδους για να κατασκευάσουν επικίνδυνο περιεχόμενο.
