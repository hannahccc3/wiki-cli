# Wiki Knowledge Base — Agent Operating Rules

## CRITICAL: You MUST use wiki-cli tools to operate the wiki. NEVER write files directly.

This wiki is managed by `wiki-cli`. All operations MUST go through its CLI or MCP tools.
Do NOT create, edit, or delete markdown files directly — always use the tools below.

## Available Tools

### MCP Tools (preferred)
Use these when running as an MCP client:
- `wiki_ingest` — Ingest a source document (auto-generates entity/concept/source pages)
- `wiki_read` — Read a page by slug
- `wiki_write` — Create a new page (frontmatter is auto-generated)
- `wiki_update_page` — Incrementally update an existing page (merges tags/related/sources)
- `wiki_delete_page` — Delete a page by slug
- `wiki_move_page` — Rename a page (change slug)
- `wiki_search` — Search page content by keyword or regex
- `wiki_query` — Query with natural language (optional LLM answer via `llm: true`)
- `wiki_list` — List all pages (optional type filter)
- `wiki_stats` — Get page counts and tag distribution
- `wiki_lint` — Check for broken links, orphans, missing fields
- `wiki_graph` — Generate knowledge graph
- `wiki_get_insights` — Get orphan pages, hub pages, coverage gaps

### CLI Commands (fallback)
```bash
wiki init              # Initialize a new wiki project
wiki ingest <file>     # Ingest a document
wiki ingest-batch <dir> # Batch ingest
wiki query <question>  # Query the wiki
wiki list              # List pages
wiki stats             # Show statistics
wiki lint              # Health check
wiki graph             # Knowledge graph
wiki delete <slug>     # Delete a page
wiki move <old> <new>  # Rename a page
wiki index --update    # Regenerate index
wiki serve             # Web UI
```

## Page Types and Directories

| Type | Directory | Purpose |
|------|-----------|---------|
| entity | `wiki/entities/` | Named things: people, organizations, models, tools, datasets |
| concept | `wiki/concepts/` | Ideas, techniques, phenomena, frameworks |
| source | `wiki/sources/` | Papers, articles, books — summary of a raw document |
| query | `wiki/queries/` | Open research questions |
| comparison | `wiki/comparisons/` | Side-by-side analysis |
| synthesis | `wiki/synthesis/` | Cross-cutting summaries and conclusions |

## Frontmatter Format (auto-generated by tools)

When you use `wiki_write` or `wiki_ingest`, frontmatter is generated automatically in this exact order:

**Entity/Concept/Query/Comparison/Synthesis pages:**
```yaml
---
type: entity
title: Page Title
created: 2026-05-02
updated: 2026-05-02
tags: [tag1, tag2]
related: [slug-1, slug-2]
sources: ["filename.md"]
---
```

**Source pages (additional fields):**
```yaml
---
type: source
title: Paper Title
created: 2026-05-02
updated: 2026-05-02
tags: [tag1, tag2]
related: [slug-1]
sources: ["filename.md"]
authors: [Author A, Author B]
year: 2024
url: "https://..."
venue: "NeurIPS 2024"
---
```

## Naming Rules

- Slugs: `kebab-case` (e.g., `gpt-4o`, `chain-of-thought`, `safety-alignment`)
- Source slugs: `author-year-short-title` (e.g., `wei-2022-cot`)
- Entity slugs: match official name (e.g., `openai`, `claude-3-opus`)
- Concept slugs: descriptive noun phrase (e.g., `random-search`)

## Cross-referencing Rules

- Use `[[slug]]` syntax for links between pages
- Every entity and concept should be listed in `wiki/index.md`
- Use `wiki_update_page` with `related` to add cross-references (merges, doesn't replace)
- Source pages MUST link to their entity/concept pages via `related`

## Workflow

### Ingesting a new source document
1. Copy the source file to `raw/sources/` (immutable originals)
2. Run `wiki_ingest` with the file path
3. Review the generated pages with `wiki_read`
4. If pages need adjustments, use `wiki_update_page` (NOT direct file editing)
5. Check health with `wiki_lint`

### Answering a question
1. Use `wiki_query` with `llm: true` to get an LLM-synthesized answer
2. If the answer is valuable, save it back as a new page via `wiki_write`
3. Good answers can become `synthesis` or `comparison` pages

### Maintaining the wiki
1. Run `wiki_lint` periodically to find broken links and orphans
2. Run `wiki_get_insights` to find coverage gaps and hub pages
3. Use `wiki_update_page` to add missing cross-references to orphan pages
4. Update `purpose.md` when the research direction evolves

## ABSOLUTE PROHIBITIONS

1. **NEVER** create or edit files in `wiki/` directly — always use wiki-cli tools
2. **NEVER** modify files in `raw/` — these are immutable source documents
3. **NEVER** write frontmatter manually — the tools generate it in the correct format
4. **NEVER** skip the `sources` field — every page must trace back to its origin
5. **NEVER** create duplicate entities — use `wiki_search` first to check if a page already exists
